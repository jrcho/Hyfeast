    hfVisualizer 상세 사용자 매뉴얼 (튜토리얼 기반 완벽 가이드)

    본 매뉴얼은 2008년부터 한국건설기술연구원에서 개발된 인하우스 하이브리드 유한요소 코드 **Hyfeast(파이 피스트)**의 전후처리 전용 툴, **hfVisualizer**의 공식 기능 가이드입니다. (Youtube 공식 튜토리얼 음성 해설을 바탕으로 모든 기능의 '실제 조작법'과 '왜 이 기능을 써야 하는지'를 10배 상세하게 풀어 썼습니다.)

    1. Hyfeast 시스템 개요와 hfVisualizer의 역할

    Hyfeast는 객체지향 프로그래밍과 절차적 프로그래밍을 혼합한 가볍고 빠른 코어로, 새로운 재료 모델이나 특수 알고리즘 테스트 목적으로 개발되었습니다. 전체 시스템은 다음과 같이 구성됩니다: - hfAnalyzer: in-core 유한요소 컴파일러 솔버 엔진 (입력: *.inp, 출력: *.odb, 바이너리 파일). - hfSectionVisualizer / hfSectionAnalyzer: 임의 형상의 복합 단면 물성치 계산 및 비선형 휨-곡률 해석 모듈. - hfVisualizer: 텍스트(.inp)로만 이루어진 유한요소 입력을 그래픽으로 조립(Pre-processing)하고, 복잡한 3D 응력 파괴 결과를 시각 브라우저로 투영(Post-processing)하는 핵심 통합 GUI 환경.

    2. 기본 화면 구성과 인터페이스 (GUI Layout)

    프로그램을 관장하는 핵심 5대 패널을 소개합니다.

    2.1 메인 캔버스 뷰포트 (Main Viewport)

    모델이 투사되는 중앙 공간입니다. - 마우스 다이내믹 뷰 (Dynamic View): 상단의 다이내믹 뷰 아이콘이 On 되어 있을 때만 마우스 드래그를 통해 부재를 회전시키거나 시점을 이동(Pan)할 수 있습니다. 뷰가 틀어지는 걸 원치 않으면 이 아이콘을 끕니다. - 마우스 휠 버튼 누르고 드래그: 화면 평행 이동(Panning). - 마우스 휠 굴리기: 화면 줌 인 / 줌 아웃. - 마우스 우클릭 드래그: 3차원 입체 회전.

    2.2 모델 트리뷰 (Model Tree)

    화면 좌측에 도킹된 탐색기로, "절점(Node), 요소(Element), 구속조건(Constraint), 하중(Load), 물성(Material/Section), 단계(Step)" 등 해석의 모든 구성 엔티티가 디렉토리 폴더 형식으로 정의(Define)되어 있습니다. 작업 내용을 한눈에 파악하는 모니터링 창입니다.

    2.3 커맨드 위젯 (Command Panel)

    중앙 우측에 배속되어, 텍스트 코멘드를 타이핑할 필요 없이 GUI 마우스 클릭만으로 Node나 Section의 생성 변수(Parametric Data)를 입력하게 해주는 제어 폼 덩어리입니다.

    2.4 셀렉션 패널 (Selection Panel)

    우측 하단에 위치하며, 복잡한 메쉬 중 **어떤 요소를 선택할지(집합 범위)**를 정의하는 부울 연산의 리모컨입니다.

    2.5 아웃풋 윈도우 (Output Window)

    중앙 하단에 위치. 모델링 오류나 해석 진행 시 뉴턴-랩슨 반복 수렴 로그 따위를 실시간 텍스트로 중계합니다.

    3. 디스플레이(Display) 편의 기능 및 시점 제어 (튜토리얼 조작기)

    3.1 뷰(View) 카메라 컨트롤 툴바

    모델을 바라보는 각도를 쾌적하게 셋업하는 기능들입니다. - Fit 줌 버튼: 모델이 너무 확대되었거나 화면 밖으로 날아갔을 때, 전체가 한눈에 들어오게 리셋 지정합니다. - Zoom to Box: 특정 힌지나 조인트 부분만 아주 세밀하게 확대해야 할 때, 이 버튼을 누르고 마우스로 사각형 범위를 '드래그 넷' 치면 그 범위만 화면에 가득 차게 줌-인 됩니다. - 정투영 버튼들 (Iso, +X, -Y ...): 3차원 아이소메트릭(Isomectric) 뷰 3종과 X,Y,Z 단면 직교 평면 뷰 버튼을 통해 즉각 화면 카메라를 단숨에 틀어줍니다. - 사용자 정의 뷰 (Set User View 1,2,3): 특정 조인트(접합부)를 보기 좋은 각도로 화면을 튼 다음 Set User View를 누르면 이 카메라 메모리가 기억됩니다. 이후 모델 여타 곳을 빙글빙글 돌리다가도 User 1 버튼 하나만 누르면 방금 전 그 얼짱 각도로 귀환합니다.

    3.2 모델 렌더링 품질 제어 (Representation)

    • Wireframe (와이어프레임): 메모리를 아끼기 위해 요소의 외곽 뼈대(Edge)만 실선으로 그립니다.
    • Surface (서피스): 면 색상을 입혀서 솔리드/쉘에 텍스처를 먹입니다 (보통 두 기능을 같이 켭니다).
    • Transparency (반투명 렌더링): 겉의 콘크리트를 투과해 내부 철근망을 검토해야 할 때 사용.
    • Element Shrink (요소 수축 검열): (중요) 수십만 개의 브릭 요소가 노드 공유 없이 겹쳐(Overlapped) 있진 않은지 체크합니다. 이 버튼을 켜면 요소들 체적을 살짝 줄여, 요소 사이에 검은 간격(틈)을 인위적으로 벌려 매쉬망 오류를 단번에 육안 감별해냅니다.
    • Section View (단면 압출 뷰): 선(Line)으로만 이뤄진 TRUSS나 BEAM 부재는 3D 체적이 없습니다. 하지만 이 기능을 켜면, Section 프라퍼티에 입력됐던 사각형, H형강 치수(폭/높이) 만큼의 덩치를 실제로 그래픽 팽창 합성을 일으켜, 빔 단면들의 결합 형토를 3D 리얼하게 시연합니다!

    3.3 로컬 뷰 및 라벨링 토글 스위치 (Display Control)

    튜토리얼 설명에 의하면 초보자들이 가장 당황하는 부분입니다. - 마커 출현의 비밀: Display Node 번호 나 Element 라벨, ECS(로컬 축 화살표) 디스플레이 아이콘을 켰는데도 화면에 번호가 안 뜨나요? 이는 메모리 부하 때문에 "선택(Select)된 객체"에게만 라벨을 띄우는 것이 디폴트이기 때문입니다. 부재를 파란색으로 선택한 후 버튼을 누르면 그제야 번호가 허공에 뜹니다. - Always On 셋업: 만약 항상 라벨이 떠 있게 하려면, [Settings] 의 디스플레이 옵션에 가서 "선택 시에만 디스플레이" 옵션을 언체크하면 됩니다 (단, 속도는 다소 느려집니다). - Shading Color (기둥/슬래브 색상 구별): By Section 버튼을 누르면, 같은 Section 단면을 가진 애들끼리 그룹 색상이 다르게 칠해집니다. 파란 기둥 사이에 빨간 기둥이 껴있다면 "아 단면 오입력이네" 하고 쉽게 오류를 파악하는 강력 툴입니다.

    4. 선택 도구 및 부위 검열 (Selection System)

    대형 교량 모델의 경우 원하는 조인트 메쉬 하나를 잡기가 전쟁일 때가 있습니다.

    4.1 부울(Boolean) 로직 채택

    우측 하단 셀렉션 패치에서 Current, Union, Intersection, XOR 등을 조절해 집합론적 중첩 구역 선택을 유도합니다. - Reset Selection의 All, Node, Element 등 버튼을 누르면 선택되어 파래진 객체 메모리들을 순간 방전 시켜 리셋합니다.

    4.2 Show / Hide 검열 은폐기

    • Show Node/Element Selection Only: 내가 고른 부위만 툭 떼어서 화면에 놔두고, 우주상의 모든 요소를 Hide 시킵니다! 복잡한 모델 수정 작업의 알파이자 오메가입니다.
    • 반대로 Hide Node/Element Selection 은 내가 선택한 외피만 투명망토를 씌워서 내부를 파고들게 합니다.

    5. 프리프로세싱 단위 실무: 위젯과 테이블

    [Preprocessing] 메뉴 하부의 각종 입력 위젯(커맨드 탭에 귀속)입니다.

    5.1 절점 및 데이터 행렬 (Node Generation)

    • 복사와 배열 (Create & Copy): 시작점 1개를 마우스로 치고, Copy Widget을 통해 \(X=3m\)마다 \(N=10\) 회전/이동 복사하여 철골 격자의 베이스룸을 깝니다.
    • Node / Element Table View: 엑셀과 같은 스프레드시트 뷰어가 윈도우로 뜹니다. 현존하는 수만 개의 노드 좌표(X,Y,Z) 데이터 텍스트 행렬을 엑셀 매크로로 복사/붙여넣기(Ctrl+V) 하여 대량 편집/수정하는 초강력 에디팅 위젯입니다.

    5.2 부재 요소망 메이킹 (Element Processing)

    • Extrude (차원 상승 압출): 평면 2D 메쉬 100개를 잡고 이 위젯의 \(Z\)방향 벡터 창에 치수값을 넣으면, 일괄적으로 3D 솔리드 덩어리 메쉬 구조체로 돌출 연마됩니다.
    • Divide / Change: 길이가 먼 단일 프레임 보를 N토막 내어 해석 곡률 해상도를 높이거나, 실수로 TRUSS로 짠 망을 BEAM 시스템으로 일괄 변환(Change)합니다.

    5.3 속성 매핑 (Assign Material & Section)

    1. 재료(Material) 위젯에서 콘크리트 압축 강도 파라미터 텍스트 박스 입력 후 라벨링.
    2. 단면(Section) 위젯에서 사각 치수 W/H 등을 세팅하고 위 재료 라벨과 연동.
    3. 뷰포트에서 발라줄 요소들을 잔뜩 Selection(파란색) 한 다음, Assign Section 메뉴의 드롭다운을 통해 방금 만든 Section 이름을 발사합니다 (일괄 맵핑).

    6. 해석 구동과 Result 포스트프로세싱 분석

    .inp 확장자를 통해 hfAnalyzer가 성공 해석을 뱉으면, *.odb 를 로드하여 공학적 평가 브라우저를 켭니다.

    6.1 시간 증분과 타임 히스토리 플레이어 (Play Panel)

    • 상단 툴의 First, Next, Last 버튼을 통해 지진 파장이나 등분 증분 하중이 누적 됨에 따라 모델이 점점 뒤틀리는 페이즈(Phase)를 시계열로 감상합니다.
    • Record (녹화) 버튼을 누르면 이 처참한 변형 과정을 AVI/MP4 영상으로 화면 캡쳐 인코딩해 저장하여 클라이언트 프레젠테이션 자료로 제출합니다.

    6.2 변형도 스케일 뻥튀기 (Scale Factor)

    정적 하중에 의한 실제 처짐량은 0.1mm 등 미세해서 안 보입니다. - Deformed 스케일 위젯창에서 이 처짐 계수에 \(\times 1,000\) 배율을 입력해주어, 눈으로 식별 가능하게 고무줄처럼 과장 렌더링을 일으켜 전체 구조 텐션 결함을 직관적으로 도출합니다.

    6.3 정밀 응력/변형률 스펙트럼 맵 (Contour View)

    • Mises 응력, 최대 인장력 한계 지수 등을 선택하면 3D 메쉬 표면이 무지개 스펙트럼 그라데이션 컬러로 도배됩니다.
    • Custom Contour Dialog에서 스펙트럼 최대 허용 한계 색상(Limit)을 강판의 항복 강도로 고정(Fix)시켜 버리면, 뻘겋게 칠해진 부위는 무조건 부서진 부재로 판단되는 빠른 검증 필터를 제공합니다.

    6.4 디지털 트윈 센서 실측비교 (DigitalTwin / Sensor Plot)

    • 모델 내 가상 센서를 달아둡니다 (Sensor Widget).
    • 우측의 Chart Control Panel 에서 해석 결과 진동 게이지 응답 데이터 Y값 선 꺾은 차트를 도출한 다음, **외부 현장에서 떼온 진짜 센서 CSV 계측 차트**를 그 위에 오버레이(합성 투영) 시켜, 모델 댐핑 세팅 값의 오류를 비교 수정(Tuning Parameter) 하는 차세대 스마트 건설 뷰어 기능입니다.
